🚀 Project Upgrade Plan: Blooket Quiz Generator 2.0
1. 개요 (Executive Summary)
본 계획서는 기존의 단순 텍스트 기반 퀴즈 생성기를 넘어, 대한민국 교육 환경(HWP, PDF, 멀티미디어 자료)에 최적화된 **'범용 게이미피케이션 학습 지원 도구'**로의 진화를 목표로 한다.

2. 핵심 개선 목표
멀티 모달 입력 확장: HWP, PDF, PPTX, MP4 등 모든 교육 자재 수용.
구조적 데이터 보존: 표(Table) 내 데이터 및 수학/과학 수식의 완벽한 컨텍스트 유지.
영상 분석 도입: 영상의 시각 정보와 음성을 결합한 복합 문제 생성.
전 과목 대응: Gemini 2.5 flash 모델을 활용한 과목별 전문 프롬프트 최적화.

3. 기술 스택 및 아키텍처 제안
A. 파일 파싱 전략 (The "Structure-First" Approach)
단순 텍스트 추출 방식에서 '문서 렌더링 후 비전 분석' 방식으로 전환한다.
HWP/PDF/PPTX: * 문제점: HWP 내부 라이브러리는 수식과 표 구조를 깨뜨리는 경우가 많음.
해결책: 문서를 PDF로 변환 후, 각 페이지를 고해상도 이미지로 스냅샷을 찍어 Gemini 2.5 flash에 전달. AI가 시각적으로 표와 수식을 직접 해석하도록 유도 (Multimodal Reasoning).

MP4 (Video): * Gemini 2.5 flash의 비디오 처리 기능을 활용. 샘플링된 프레임(이미지)과 오디오 트랙을 함께 분석하여 "영상 속 실험의 결과는 무엇인가?"와 같은 시각적 추론 문제 생성.

B. 데이터베이스 및 서버 아키텍처
클라이언트: 사용자 개인 API Key 입력 및 로컬 스토리지 저장.
서버 (Backend): HWP 파일을 PDF로 변환하거나 영상을 프레임 단위로 처리하기 위한 가벼운 Python 기반 API 서버(FastAPI 등) 필요.

4. 단계별 상세 개발 계획
1단계: 멀티 파서(Multi-Parser) 구축
pdf2image, python-pptx 등을 활용하여 업로드된 파일을 이미지 스트림으로 변환.
HWP의 경우, 서버사이드에서 Pandoc 혹은 전용 라이브러리를 통해 구조 보존 파싱 시도.

2단계: 과목별 전문 프롬프트 엔지니어링 (Persona Tuning)
Gemini 2.5 flash를 컨트롤하기 위한 시스템 프롬프트 세분화:
수학/과학: LaTeX 문법 준수 및 풀이 과정의 논리적 검증.
음악/체육: 이미지/영상 속의 동작이나 악보의 시각적 특징 추출.
국어/영어: 지문의 난이도(Lexile 지수 등)에 따른 어휘 수준 조절.

3단계: Blooket CSV 매핑 레이어 고도화
추출된 문제를 Blooket 표준 양식(Column A-G)에 맞게 강제 정렬하는 Validator 로직 추가.
시간 제한(Time Limit): 문제의 난이도와 지문 길이에 따라 AI가 적정 시간을 자동 계산하도록 설정.

5. 예상되는 병목 구간 및 대책 (Risk Management)
토큰 소모 : 고해상도 이미지 및 영상 분석 시 토큰 사용량 급증. 사용자 개인 API Key 사용 가이드를 제공하고, 요금 모니터링 기능 추가
HWP 호환성 : 한글 문서 특유의 폐쇄적 구조 및 암호화 문서. EasyOCR 또는 Gemini Vision을 통한 이미지 기반 분석으로 우회
수식 오류 : 분수, 루트 등 복잡한 수식이 깨질 가능성. 수식을 Markdown 또는 LaTeX으로 변환하도록 프롬프트 지시어 고정

6. 결론 및 제언본 프로젝트는 **'기술적으로 가장 까다로운 한국형 교육 문서'**를 정복하는 데 핵심이 있습니다. 단순히 모델을 올리는 것보다, AI가 문서를 '사람처럼 읽게' 만드는 이미지 기반의 분석(Visual Reasoning) 방식을 적극 채택해야 합니다.
