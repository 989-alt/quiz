# 블로그 자동화 가이드

텔레그램으로 주제를 보내면 Gemini가 리서치하고, 이미지를 수집하고, Gemini 2.5 Pro가 글을 작성해서 Blogger에 발행 + HTML 파일로 저장하는 자동화 파이프라인입니다.

---

## 파이프라인 흐름

```
텔레그램 명령어 입력
    ↓
researcher.js   Gemini 2.5 Flash + Google Search 그라운딩 → 주제 리서치
    ↓
media-finder.js Pexels API (중복 방지 + 랜덤 페이지) → 이미지 수집
    ↓
claude.js       Gemini 2.5 Pro → SEO 최적화 블로그 글 + 이미지 삽입
    ↓
pipeline.js     템플릿 적용 → HTML 렌더링
    ↓
blogger.js      Blogger API v3 → 발행 또는 초안 저장
output/         HTML 파일 저장 → 티스토리 수동 업로드
    ↓
텔레그램        완료 메시지 + URL/파일 경로 전송
```

---

│   ├── telegram-bot.js    텔레그램 봇 진입점

│   ├── index.js           CLI 진입점
│   └── news-fetcher.js    RSS 뉴스 수집 (레거시)

## 환경 설정

### 필수 API 키 (.env)

```env
# Claude API (Blogger RSS 레거시 모드용)
ANTHROPIC_API_KEY=sk-ant-...

# Blogger 블로그 ID (blogger.com 대시보드 URL에서 확인)
BLOGGER_BLOG_ID=6218265260824404297

# 사용 모델
CLAUDE_MODEL=claude-opus-4-6

# 텔레그램 봇 토큰 (@BotFather에서 /newbot으로 발급)
TELEGRAM_BOT_TOKEN=...

# Gemini API (리서치 + 글 작성)
# https://aistudio.google.com/apikey
GEMINI_API_KEY=...

# Pexels 이미지 API
# https://www.pexels.com/api/new/
PEXELS_API_KEY=...

# Google Custom Search (선택, CC 라이선스 이미지 검색용)
# Google Cloud Console에서 Custom Search API 활성화
# https://programmablesearchengine.google.com 에서 CX 발급
GOOGLE_CSE_KEY=
GOOGLE_CSE_CX=
```

### Google OAuth 초기 설정 (최초 1회)

1. Google Cloud Console → APIs & Services → Library에서 **Blogger API v3** 활성화
2. Credentials → Create Credentials → **OAuth 2.0 Client ID** (Desktop app)
3. 다운로드한 JSON을 `credentials.json`으로 저장
4. 인증 실행:
   ```bash
   node src/auth.js
   ```
5. 브라우저에서 Google 계정 로그인 → 권한 허용 → 자동 완료
6. `token.json` 생성 확인

## 실행 방법

### 텔레그램 봇

```bash
npm run bot
# 또는
node src/telegram-bot.js
```

#### 명령어

| 명령어 | 동작 |
|--------|------|
| `/write <주제>` | Blogger 초안 저장 + HTML 파일 생성 |
| `/publish <주제>` | Blogger 즉시 발행 + HTML 파일 생성 |
| `/file <주제>` | HTML 파일만 생성 (Blogger 미발행, 티스토리용) |
| `/status` | 현재 파이프라인 진행 상태 확인 |
| `/help` | 사용법 안내 |

#### 예시

```
/write AI safety in education
/publish The future of quantum computing
/file 내가 만든 블로그 자동화 앱 소개
```

### CLI (터미널)

```bash
# 주제 기반 파이프라인 (초안)
node src/index.js --topic "AI safety" --draft

# 주제 기반 파이프라인 (즉시 발행)
node src/index.js --topic "quantum computing" --publish

# RSS 뉴스 파이프라인 (레거시)
node src/index.js --rss --hours 24

# 도움말
node src/index.js --help
```

## Google Blogger 자동화

### 발행 방식

- **초안 (`--draft`)**: Blogger 대시보드에 초안으로 저장. 수동으로 검토 후 발행.
- **발행 (`--publish`)**: 바로 공개 발행. URL 즉시 확인 가능.

### 확인 경로

- https://www.blogger.com → 블로그 선택 → 게시물 → 초안/발행됨

### 구조 권장 (라벨 기반)

Blogger는 카테고리 대신 **라벨(Labels)**을 사용합니다. 파이프라인이 자동으로 주제에 맞는 라벨을 생성합니다.

| 내용 유형 | 자동 생성 라벨 예시 |
|---------|-----------------|
| AI 최신 동향 | `AI`, `News`, `Machine Learning` |
| 프로그램 소개 | `Project`, `App`, `Tool`, `Tutorial` |
| 활용 기록 | `AI Usage`, `Guide`, `Review` |

### OAuth 토큰 만료 시

`token.json` 삭제 후 재인증:
```bash
del token.json
node src/auth.js
```

## 티스토리 수동 업로드

티스토리 Open API가 서비스 종료되어, **파일 저장 방식**으로 운영합니다.

### 블로그 카테고리 구조

```
티스토리 블로그
├── 📁 내가 만든 프로그램들
│   └── 각 프로그램 소개 + 접속 링크 + 사용법
├── 📁 AI 뉴스
│   └── 최신 AI 동향 (Gemini 리서치 기반)
└── 📁 AI 활용 기록
    └── AI 도구 활용 경험, 팁, 리뷰
```

### 업로드 절차

1. **글 생성**
   ```
   텔레그램: /file 내가 만든 블로그 자동화 앱
   ```
   또는
   ```bash
   node src/index.js --topic "주제" --draft
   ```

2. **파일 위치 확인**
   ```
   blog-automation/output/2026-03-08-주제-제목.html
   ```

3. **브라우저에서 미리보기**
   - HTML 파일을 더블클릭 → Chrome에서 열림
   - 레이아웃, 이미지 삽입 위치 확인

4. **티스토리에 붙여넣기**
   - 티스토리 글쓰기 → **[기본모드]** 오른쪽 상단 `HTML` 버튼 클릭
   - HTML 파일의 `<body>` 안 내용 전체 복사 (Ctrl+A → Ctrl+C)
   - 티스토리 HTML 에디터에 붙여넣기 (Ctrl+V)
   - 카테고리 선택 → 발행

### 카테고리 분류 기준

| 주제 특성 | 카테고리 |
|---------|---------|
| news, update, report, breaking 포함 | AI 뉴스 |
| project, app, tool, program, built 포함 | 내가 만든 프로그램들 |
| 그 외 모든 주제 | AI 활용 기록 |

## 이미지 관련

### 소스

1. **Pexels API** (기본): 로열티 프리, 상업적 사용 가능
   - 중복 방지: 세션 내 사용된 사진 ID 추적
   - 다양성: 랜덤 페이지(1~5) + 키워드 조합 변형

2. **Google Custom Search** (선택): CC 라이선스 이미지
   - `GOOGLE_CSE_KEY`, `GOOGLE_CSE_CX` 설정 시 자동 사용
   - Creative Commons 라이선스만 필터링

### 저작권 표기

생성된 HTML 하단에 자동으로 이미지 출처가 표기됩니다:
```
사진 출처: John Doe (Pexels), Jane Smith (Pexels)
```

## 트러블슈팅

| 문제 | 해결 방법 |
|------|---------|
| 봇이 응답 안 함 | 다른 터미널에서 동일 토큰으로 봇이 실행 중인지 확인. `taskkill /F /IM node.exe` 후 재시작 |
| Blogger 인증 오류 | `token.json` 삭제 후 `node src/auth.js` 재실행 |
| Gemini API 오류 | `GEMINI_API_KEY` 확인. Google AI Studio에서 키 상태 확인 |
| 이미지가 0개 | `PEXELS_API_KEY` 확인. 키가 올바른지 Pexels 대시보드에서 확인 |
| 같은 이미지 반복 | `GOOGLE_CSE_KEY` + `GOOGLE_CSE_CX` 설정으로 Google CSE 이미지 추가 |
